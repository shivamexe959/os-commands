🔗 MATRIX CHAIN MULTIPLICATION
Dynamic Programming · Optimal Parenthesization
⏱ Time: O(n³) 💾 Space: O(n²) Technique: DP

Matrix Chain Multiplication finds the optimal order to multiply a sequence of matrices, minimizing total scalar multiplications.

dp[i][j] = min over all k: dp[i][k] + dp[k+1][j] + p[i-1]·p[k]·p[j]
Base case: dp[i][i] = 0

where p[] contains matrix dimensions. Matrix Aᵢ has dimensions p[i-1]×p[i].

💡 Exam Tip: Always fill the DP table by chain length (l=2,3,...,n), not row by row. Write both the cost table AND the split table for full marks.

🪙 COIN CHANGE PROBLEM
Greedy & Dynamic Programming · Minimum Coins
⏱ DP: O(n·W) 💾 Space: O(W) Techniques: Greedy + DP

Coin Change finds the minimum number of coins to make a given amount. Greedy works only for canonical coin systems; DP always gives optimal result.

dp[0] = 0
dp[i] = min(dp[i - coin] + 1) for all coins ≤ i
dp[i] = ∞ if not reachable
💡 Exam Tip: Always show the greedy fails for non-canonical systems (e.g., coins {1,3,4}, amount=6 → greedy gives 4+1+1=3 but DP gives 3+3=2).

💼 JOB SCHEDULING PROBLEM
Greedy Algorithm · Maximize Profit with Deadlines
⏱ Time: O(n²) 💾 Space: O(n) Technique: Greedy

Job Scheduling: Given jobs with deadlines and profits, schedule at most one job per slot to maximize total profit. Greedy: sort by profit descending, assign each job to latest available slot ≤ its deadline.

Sort jobs by profit: p₁ ≥ p₂ ≥ ... ≥ pₙ
For each job: find slot t = min(deadline, slots) going backward
If slot t is free → schedule job there
💡 Exam Tip: Always show the slot array. After sorting by profit, go job-by-job and fill the latest available free slot ≤ deadline.

🗺️ DIJKSTRA'S SHORTEST PATH
Greedy + Priority Queue · Single Source Shortest Path
⏱ Time: O((V+E) log V) 💾 Space: O(V) Technique: Greedy PQ

Dijkstra's algorithm finds the shortest path from a source node to all other nodes in a weighted graph with non-negative edges.

dist[src] = 0, dist[v] = ∞ for all v ≠ src
Relax: if dist[u] + w(u,v) < dist[v] → dist[v] = dist[u] + w(u,v)

Uses a priority queue (min-heap) to always process the node with smallest tentative distance.

💡 Exam Tip: Show the distance table evolution at each step. Mark finalized nodes clearly. Show the priority queue state.

🌐 FLOYD-WARSHALL ALGORITHM
Dynamic Programming · All-Pairs Shortest Paths
⏱ Time: O(n³) 💾 Space: O(n²) Technique: DP

Floyd-Warshall computes shortest paths between ALL pairs of vertices in a weighted graph, including negative edges (but no negative cycles).

D⁰: initial distance matrix
Dᵏ[i][j] = min(Dᵏ⁻¹[i][j], Dᵏ⁻¹[i][k] + Dᵏ⁻¹[k][j])
Apply for k = 1, 2, ..., n
💡 Exam Tip: GTU loves Floyd-Warshall. Show D⁰, D¹, D², ..., Dⁿ tables clearly. Mark ∞ for no direct edge. Highlight cells that improve in each iteration.
Input Adjacency Matrix (∞ = 999, 0 = diagonal)
